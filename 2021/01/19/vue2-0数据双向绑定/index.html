vue2.0数据双向绑定

是采用数据劫持结合发布者-订阅者模式的方式，通过Object.defineProperty()来劫持各个属性的setter，getter，在数据变动时发布消息给订阅者，触发相应的监听回调。

缺点

  1. 在Vue中，Object.defineProperty无法监控到数组下标的变化，
  2. Object.defineProperty只能劫持对象的属性,因此我们需要对每个对象的每个属性进行遍历。

vue3.0数据双向绑定

优点

  1. 可以劫持整个对象，并返回一个新对象
  2. 有13种劫持操作

vue2.0与3.0的区别

·  3.0比2.0 快2倍

·  3.0去掉了filter, 没有beforeCreate created,用setup取代

· reactivity是可以单独作为库使用的 

· 单独功能可以抽离 取代了mixin  优于mixin 解决上下反复横跳

· 支持多个子节点 fragment

· setup里没有this

· Proxy实现响应式不需要set delete  兼容性并不好

· 响应式方面 性能得到很大提升 不用初始化的时候就递归遍历属性

· 响应式不区分数组和对象

· 3.0兼容IE12以上

· composition api 可以和 options API 同时存在
